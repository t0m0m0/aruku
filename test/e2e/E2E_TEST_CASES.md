# E2E テストケース一覧

aruku アプリの主要なユーザーフローをカバーするウィジェットテストのテストケース仕様。  
実装ファイルは `test/e2e/` 配下。

> **ナビゲーション基盤（#165 以降）**
> 画面遷移は go_router が権威。`test/e2e/support/e2e_helpers.dart` の `appWidget`
> は本番と同じ `goRouterProvider` を積んだ `MaterialApp.router` を返す。
> 戻る操作は実 Navigator の pop（`tester.binding.handlePopRoute()`）で検証でき、
> back 分岐の網羅は `back_navigation_flow_test.dart` を参照。
> 遷移アニメ（220ms）のため、新画面を検証する前に遷移分の pump が必要な箇所がある。

## 2. ルート検索フロー
**ファイル:** `route_search_flow_test.dart`

### TC-SRCH-01: 目的地が未設定のとき「目的地を選ぶ」ボタンが表示される

|---|---|

|---|---|
| 前提 | 目的地設定済み、ルートサービス = 応答保留中 |
| 操作 | 「ルートを検索」をタップ |

### TC-SRCH-05: 結果画面で目的地名が表示される
| 項目 | 内容 |
|---|---|

| 操作 | 戻る操作（システム back / `go(Screen.home)`）でホームへ戻る |

## 未カバーのフロー（将来対応候補）

| フロー | 概要 | 難易度 |
|---|---|---|
| 目的地検索 UI | SearchScreen で候補タップ → 目的地確定 | 中（PlacesService フェイク要） |
| 出発地変更 | SearchOrigin で現在地以外を設定 | 中 |
| 時刻変更フロー | ピッカーで出発/到着を変更 → 予算が更新される | 低 |
| 設定 → 通知トグル → ホーム戻り | 変更後の状態がホームに影響しないことを確認 | 低 |
| 位置情報拒否時の検索 | `LocationDenied` 状態でも検索を試みる | 低 |
| オフルート自動再検索 | GPS が経路から外れると再検索が走る | 高（位置ストリームモック要） |
